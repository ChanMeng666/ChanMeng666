# Chan Meng — Personal Brand & Design System

> *Subtraction for life, addition for thought.*

A personal visual + verbal design system for Chan Meng — AI Agent Architect, Full-Stack Engineer, and minimalist living advocate based in Auckland, NZ. This document is the **human-readable counterpart** to `data/brand.yaml` (machine-readable single source of truth). Token tables between `<!-- AUTO:* -->` fences are regenerated by `npm run build:brand`; everything else is hand-edited.

The **v2.0.0 "Caldera" re-skin** clones the [Caldera "Pixelated Cyber-Playground"](https://caldera.xyz) design system: a high-contrast digital-arcade identity. The Caldera-format export (DTCG tokens, Tailwind `@theme`, plain `:root`, and a generated `DESIGN.md`) lives in [`dist/brand/`](../../dist/brand/).

---

## 1. Overview

This system serves four cross-surface needs:

- **GitHub README** — the canonical landing page. Markdown + own-decoration SVGs only (pixelated orange/violet banners via `gradient-svg-generator`).
- **CV** — print-tuned Typst PDF + JSON-LD + agent-readable text. "Tuned-for-print" Caldera: compact ultrabold name/headers, orange accent rules, recruiter/ATS-safe density.
- **Web** — future personal site at `chanmeng.org`. Consumes `dist/brand/tokens.json` (or the Caldera-format `design-tokens.json` / `theme.css`) as its source of truth.
- **Social/Newsletter** — single-screen surfaces (LinkedIn services cards, newsletter headers, share images).

The brand is a **pixelated cyber-playground**: a muted basalt-grey canvas backdrops vivid pixelated orange/violet forms and ultra-bold compact display type. Cards and buttons use generous rounded corners for a friendly, toy-like solidity. Color is used sparingly for impact; hierarchy comes from **solid color blocks**, and the system stays **flat** — no shadows, no elevation gradients. Every change should make the system *smaller*, not bigger.

## 2. Identity & Voice

**Name:** Chan Meng
**Tagline:** *Building at the intersection of AI, cultural technology, and women's health.*
**Philosophy:** *Subtraction for life, addition for thought.*
**Audiences:** Founders evaluating an AI agent architect / fractional CTO. Recruiters in AI / full-stack / FemTech. Engineers and learners following the open-source craft. Chinese-speaking developer communities (中文社区). Te Ao Māori partners and cultural-technology collaborators.

### Voice Principles

1. **Architecture-grade vocabulary** — engineering nouns (*orchestration, dispatch, hub-and-spoke*) over marketing adjectives (*powerful, robust, world-class*). Recruiters who matter recognise the nouns.
2. **Evidence-first** — every claim links to one proof point (PR, repo, live URL, paper). Adjectives without links get cut.
3. **Subtraction for life, addition for thought** — at most one orange accent per visual unit. Remove decoration until what is left is the idea. (The surface is loud; the discipline is not.)
4. **Cross-cultural fluency without translation** — te reo Māori, 中文, English appear in their own forms; italicised once, then treated as first-class vocabulary. No parenthetical glosses unless the audience demonstrably needs them.
5. **Pace by white space** — visual rhythm beats density. The 61.8 / 38.2 ratio from the CV is canonical; default to it elsewhere.
6. **Never apologise for restraint** — no "minimal-but-", no "sorry it's plain". The confidence *is* the statement.

### Vocabulary

**Prefer:** orchestration · dispatch · hub-and-spoke · agentic · evidence-first · shipped · open-sourced · production · end-to-end
**Avoid:** leverage · robust · world-class · best-in-class · cutting-edge · synergy · rockstar · ninja

**Rules**

- Templates consume `--color-canvas-page`, `--color-ink-primary`, `--color-accent-primary` etc. — never the raw palette.
- The canvas is **never pure white** — `#E2E2DF` basalt is the page floor; `#F7F6F2` ash carries cards.
- **Orange is the one voltage.** Use it as a block background or accent rule — never as small body or link text (it fails contrast). Links and labels are ink.
- **Violet is decorative-only** — large background shapes and decorative cards, never text or UI.
- **Glare (yellow)** is a highlight overlay / graphic accent — never a text or fill color for content.
- A WCAG AA contrast gate (`scripts/build-brand.mjs` · `contrastRatio`) fails the build if a text role resolves to violet or orange, or if ink-on-canvas regresses.

## 4. Typography

Two voices plus a mono house-extension. Display is ultra-bold compact (**Bebas Neue**); body is **DM Sans 500**; mono is reserved for data signatures.

<!-- AUTO:typography-families -->
| Family | Stack | Weights | Role |
|---|---|---|---|
| `--font-display` | Bebas Neue, Anton, Oswald, Impact, sans-serif | 400 | Display, headings, and stat numbers — ultra-bold compact, tight. |
| `--font-body-sans` | DM Sans, Inter, Segoe UI, system-ui, sans-serif | 400, 500, 700 | Body, navigation, labels — DM Sans 500, the workhorse. |
| `--font-mono` | JetBrains Mono, Consolas, ui-monospace, monospace | 400, 500 | Data signatures only — datestamps, paths, version numbers. House extension (Caldera has no mono). |

| `--text-mono-*` | 13px | 8pt | 1.5 | 0 | 400 | mono |
<!-- /AUTO:typography-scale -->

**Principles**

- Display (Bebas Neue) is the name, every heading, and stat numbers — ultra-bold compact, `0.02em` tracking on display only.
- Eyebrow (`--text-eyebrow-*`) is an uppercased DM Sans label; the display face is uppercase by nature.
- Mono is reserved for data signatures: datestamps, paths, version numbers.
- Body never below 16 px on web or 9 pt in print; DM Sans 500 is the workhorse.
- Flat by default: hierarchy comes from solid color blocks, not shadows or gradients.

**License notes.** Bebas Neue, DM Sans, and JetBrains Mono are all OFL (free for web embed and self-host). They are **vendored at [`cv/fonts/`](../../cv/fonts/)** and loaded via `typst compile --font-path cv/fonts`, so the CV PDF renders deterministically on any machine / CI regardless of the OS font book. On the web, Google Fonts serves the same families; Segoe UI / Consolas remain system fallbacks.

## 8. Decorative Signatures

Five micro-signatures Chan owns — every one built on assets she controls (`gradient-svg-generator`, the monogram, OpenType features). External decoration services (shields.io, trophies, wakatime, streak counters) are **forbidden**.

### 8.1 Hero band (`heroBand`)

`gradient-svg-generator` template `pixel-art-retro` with `gradientType=pixelArt`, recoloured to the Caldera palette (`color0=FC5000 color1=524AE9 color2=F5F28E color3=070607`), h=200, dur=4s. The README's top-of-page identity statement.

- **Use:** top of any long-form surface, once per page.
- **Do not use:** internal sections, multiple per page, cards.

### 8.2 Accent editorial eyebrow (`goldEyebrow`)

`--text-eyebrow-*` in ink with a short orange `pixelArt` rule above a display heading. (Key name retained for back-compat; it is now orange, not gold.)

- **Use:** above any section that opens with the display face. Above the hero subhead.
- **Do not use:** inline body. More than one orange mark per visual unit.

### 8.3 Mono datestamp footer (`monoDatestamp`)

`--text-mono-*` 13 px in ink. Format: `last updated · YYYY-MM-DD · build N`.

- **Use:** footer of every long-form surface (README, CV, brand-system, future webpages).
- **Do not use:** short surfaces (social cards, single-screen pages).

### 8.4 Monogram dotted divider (`monogramDivider`)

`chan-monkey-logo-black.svg` @ 24 px between two `--color-rule-hairline` dotted lines.

- **Use:** between major narrative shifts (e.g. between *Featured work* and *Open source*). Replaces bare `---` between sections.
- **Do not use:** between sibling cards or list items.

### 8.5 Pill gradient rotation (`pillRotation`)

The CTA / badge vocabulary — `pixel-art-retro` + `gradientType=pixelArt` recoloured to two-tone Caldera palettes per role, centrally tunable from `brand.yaml signatures.pillRotation.templates`.

- **Use:** CTAs, social badges, action pills.
- **Do not use:** inside body prose. As decoration. Outside Chan's own tools.

## 10. Do's and Don'ts

| Do | Don't |
|---|---|
| Use `--color-canvas-page` (`#E2E2DF`) as the page background. | Use pure white as a page background. |
| Use `--color-surface-accent` (orange) as a block fill / accent rule. | Use orange as small body or link text (it fails contrast). |
| Apply the display face (Bebas Neue) to the name, headings, and stat numbers. | Set headlines in a generic sans. |
| Keep `--color-surface-decor` (violet) to large background shapes. | Use violet as text or a UI color. |
| Keep cards at `--radius-card` (40 px) and pill buttons at `--radius-pill`. | Mix radii inside one component. |
| Rely on solid color blocks for hierarchy. | Add shadows or elevation gradients. |
| Render numeric signatures (datestamps, versions) in `--font-mono`. | Set datestamps in the body sans. |

## 11. Iteration Guide

**Adding a token.** Edit `data/brand.yaml`. Run `npm run validate` to catch schema mistakes. Run `npm run build:brand` to run the WCAG contrast gate and regenerate `dist/brand/tokens.{css,json}`, the Caldera-format `design-tokens.json` / `theme.css` / `variables.css` / `DESIGN.md`, `cv/tokens.typ`, the token tables in this document, and `public/brand-system.html`.

**Changing a hex.** Same as above — but read the diff carefully. A color change ripples through every emitted artifact, and the contrast gate will block the build if it puts unreadable color where copy goes. The CV PDF regenerates on the next `pwsh cv/build.ps1`.

**Proposing a new signature.** Add it under `signatures.*` in `brand.yaml`. Update the relevant `$defs` in `schema/brand.schema.json` if it doesn't match an existing shape (e.g. `bannerSpec` now carries optional `gradientType` + `colors`). Document it in §8 with a *use when* / *do not use when* pair.

**The meta-rule:** every change should make the system *smaller*, not bigger. If the next iteration adds three new tokens and removes none, stop.

## 12. Known Gaps

What's out of scope, intentionally:

- **Dark mode.** Light-only for now. A `color.semantic.dark.*` namespace can be added when there's a real surface that needs it.
- **Motion library.** Tokens exist (`--duration-*`, `--easing-*`) but there are no `@keyframes` primitives. When the future webpage needs them, *it* defines them.
- **Component library.** No React Button / Card / Pill primitives in this repo. The web consumer (chanmeng.org) builds its own against `tokens.json` / `design-tokens.json`.
- **Icon system.** `/public/tech-logos/` is an asset-management problem, not a brand-system one. Don't conflate.
- **Multi-brand support.** No sub-brand tokens for FemTech / Whiri AI / etc. Sub-brands fork the tokens and override.
