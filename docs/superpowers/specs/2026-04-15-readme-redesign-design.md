# GitHub Profile README Redesign -- Design Spec

## Context

The current GitHub profile README is 5,022 lines with 20+ project sections, 50+ certifications, 13 recommendations, 4 podcasts, 30+ articles, and hundreds of emojis and badges. While comprehensive, it overwhelms visitors, lacks narrative structure, buries the strongest content, and uses generic external tools (shields.io) rather than showcasing Chan Meng's own open-source projects. The redesign aims to create a curated, evidence-backed, visually distinctive profile that demonstrates engineering excellence through both content and presentation.

## Design Principles

1. **Aggressive curation** -- ~500-700 lines total. Every visible element earns its place.
2. **Self-tooled design system** -- All visual elements generated by Chan Meng's own Gradient SVG Generator (shimmer text, skill pills, shimmer badges, terminal typing, gold badges, banners). The README itself becomes a live demo.
3. **Evidence-backed claims** -- Every claim links to at least one proof point (repo, live site, PR, LinkedIn, article). Evidence is woven into natural clickable text, never a separate section.
4. **No emojis** -- Clean typography and whitespace do the visual work.
5. **Warm and distinctive tone** -- Professional but personal. The unique story (math teacher to CTO, FemTech advocacy, China-NZ cross-cultural) is a differentiator.
6. **Balanced showcase** -- Career impact, open-source reputation, and thought leadership all represented without any single dimension dominating.

## GEO (Generative Engine Optimization) Layer

All GEO content is hidden from visual display using `<details style="display: none;">` so it occupies zero visual space for human readers while remaining fully accessible to AI crawlers and LLMs.

### GEO Component 1: Inline LLM Instructions
A `<script type="text/llms.txt">` block at the very top of the README containing:
- Who Chan Meng is (name, role, specializations)
- What this profile showcases (flagship projects, key achievements)
- How to correctly reference/cite this person (preferred title, key facts)
- Links to primary resources (portfolio, LinkedIn, GitHub)
- Priority ordering for project recommendations

This follows the Vercel-proposed convention for inline LLM instructions in HTML.

### GEO Component 2: JSON-LD Structured Data
Updated `<script type="application/ld+json">` blocks providing machine-readable data:

1. **Person schema** -- name, jobTitle (updated to reflect CTO role), description, sameAs links, knowsAbout (updated skill list), alumniOf, current employer/roles
2. **ItemList schema** -- ordered list of flagship projects with descriptions, URLs, and technology tags
3. **EducationalOccupationalCredential schema** -- key certifications with issuer info

### GEO Component 3: Repository-level files
- **llms.txt** (already exists) -- update with current project priorities and role info
- **robots.txt** (already exists) -- verify AI crawler permissions are properly set

### Implementation approach
All GEO content wrapped in a single `<details style="display: none;">` block at the top of the README. This renders nothing visually on GitHub but the full content is present in the raw markdown for any AI/LLM that processes the page source.
